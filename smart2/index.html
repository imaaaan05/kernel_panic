Errores Frecuentes y Soluciones
Si algo no va bien, aquí está la respuesta
🐛 Troubleshoot
'streamlit' is not recognized
Causa: El entorno virtual no está activo.
Fix:
venv\Scripts\activate source venv/bin/activate
ModuleNotFoundError: No module named 'openpyxl'
Causa: Falta instalar dependencias.
Fix:
pip install openpyxl xlrd
FileNotFoundError: database_clean.xlsx
Causa: El dataset no está en la carpeta del proyecto.
Fix: Copia el archivo Excel/CSV junto a app.py, o súbelo desde el sidebar de la app.
Port 8501 is already in use
Causa: Ya hay otra app Streamlit corriendo.
Fix:
streamlit run app.py --server.port 8502
No alerts generated
Causa: El dataset no tiene suficiente historial (mínimo 2 compras por par Cliente/Familia para calcular Days_Between).
Fix: Verifica que la columna Fecha tiene fechas correctas y hay registros repetidos por cliente.
KeyError: 'Id.Cliente' / 'Familia_H'
Causa: Las columnas de tu Excel tienen nombres ligeramente distintos (espacios, mayúsculas…).
Fix: Abre el Excel, comprueba los encabezados exactos y ajusta el rename_map en app.py línea ~85.

🏗️ Pipeline Architecture

📂
Step 1

Data Prep & Mapping

Load XLSX/CSV, rename columns to English, classify Analytical Block (Commodities vs Technical), compute Loyalty Factor, aggregate at Client×Date×Family grain, calculate Days_Between.

📐
Step 2

Statistical Baselines

Per-pair Mean & Std for Quantity and Days_Between. Cold-start fallback: < 3 purchases → use global Product Family stats.

🚨
Step 3

Alert Logic

Rule A (Capture Window) for Promiscuous Commodity clients. Rule B (Churn Risk) for Technical products — time delay + volume drop signals.

🎯
Step 4

Priority Score

Raw score = Avg Transaction Value × delay multiplier. Min-Max normalised to 1–100 → Nivel de Riesgo column.

🖥️
Step 5

Streamlit UI

Interactive dashboard with KPI cards, sidebar filters, editable alert table with SelectboxColumn, feedback CSV export, and drill-down expanders.

⚖️ Decision Rules — Full Explainability

Rule A · Commodities

🟠 Ventana de Captura

Block == 'Commodities' AND Loyalty == 'Promiscuous' AND Days_Since_Last >= Mean_Days Opportunity = Client_Potential − Average_Monthly_Spend

Client should have reordered by now. Likely buying from a competitor. Classic window for competitor displacement.

Rule B · Technical

🔴 Churn Risk — Two Signals

Expected_Days = Mean_Days × (Last_Qty / Mean_Qty) Time → Days_Since_Last > Expected_Days + 1.5 × Std_Days → "Retraso anómalo en tiempo" Volume → Last_Qty < Mean_Qty − 1.0 × Std_Qty → "Caída drástica de volumen" Opportunity = Average_Monthly_Spend

Bulk-adjusted time threshold prevents false alarms on large orders. Volume signal catches early disengagement.

Step 4 · Score

🎯 Nivel de Riesgo (1–100)

Raw = Avg_Transaction_Value × (1 + Days_Delayed / Mean_Days) Nivel_Riesgo = 1 + 99 × (Raw − min) / (max − min) → rounded to 0 decimals

Higher monetary value + longer overdue delay = higher urgency score. Perfectly min-max bounded for sales team prioritisation.
